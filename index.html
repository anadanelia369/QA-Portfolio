🐛 Notable Bug Found

GET /products?limit=3 returns 20 products instead of 3

console.log(pm.response.json().length);
// → 20 (expected: 3)
// limit param not implemented

Verified via Postman Console. Demonstrates parameter validation testing and backend defect identification.

pm.test("Status code is 200", () => {
pm.response.to.have.status(200);
});

pm.test("Response has products", () => {
const json = pm.response.json();
pm.expect(json).to.be.an('array');
pm.expect(json.length).to.be.greaterThan(0);
});
// ✓ PASSED · ✓ PASSED

👤 About

Observation is my API

In software systems, an API is an interface — a way to communicate with a system and understand how it behaves. In my testing work, observation becomes that interface.

Before writing a bug report, I start by watching how a system behaves under real user interaction — paying attention to the small details, the moments when expected behavior and actual behavior begin to diverge.

Testing, for me, is not only verification. It is exploration. I'm drawn to exploratory testing approaches and the tour-based ideas of James Whittaker.

Currently developing Java + Selenium automation skills at Quality Academy. ISTQB FL planned June 2026.
