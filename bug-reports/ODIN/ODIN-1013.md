# ODIN-1013

Title                          : System Back Button Breaks Trip Flow
Company: ODIN
Device/Browser: Android 14 / Realme 10
Environment: UAT
Severity: High
Status: Done
Type: Functional

Severity: `High,`Reproducibility: `100%`

---

**`Preconditions:`**

- User is logged into ODIN App as Traveler
- At least two Trips exist in the Trips list

---

**`Steps to Reproduce:`**

1. Open ODIN App
2. Navigate to **Trips** tab
3. Open **Trip A** — view its details
4. Press Android **System Back button**
5. Open **Trip B** — view its details

---

**`Actual Result:`**

Trip B screen loads and displays data from Trip A — wrong trip information is shown after using System Back navigation.

---

**`Expected Result:`**

Each Trip must display its own correct data regardless of navigation history. System Back button must not cause data bleed between Trip records.

---

**`Root Cause Guess:`**

Trip detail screen likely does not reset its state on navigation — fragment backstack retains previous Trip's data and re-renders it when a new Trip is opened via back navigation. Possible issue with ViewModel not clearing on navigation events.

---

**`Impact & Rationale:`**

Traveler may view and act on incorrect Trip data — including wrong dates, destinations, or weight limits. In a P2P delivery context this could lead to accepting the wrong order or missing a delivery. High severity because it silently shows wrong data with no error indication to the user.
