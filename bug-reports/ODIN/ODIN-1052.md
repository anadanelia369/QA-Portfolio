# ODIN-1052

Title                          : Phone Number Input Focus Jumps to City Field on First Tap  
Company: ODIN
Device/Browser: Android 14 / Realme 10
Environment: UAT
Severity: Medium
Status: Done
Type: UX/Focus

**Severity:** `Medium` **Reproducibility:** `100%`

`Preconditions:`

- User is logged into ODIN App with a new account
- Finish Setup flow is accessible

---

**`Steps to Reproduce:`**

- Open ODIN App
- Navigate to **Finish Setup → Account Verification** screen
- Tap on the **Phone Number** input field

---

**`Actual Result:`**

On first tap, focus automatically jumps to the **City** input field. User must tap the Phone Number field a second time to place the cursor and begin typing.

**`Expected Result:`**

Tapping the Phone Number field should immediately activate that field — focus must not jump to City field.

---

**`Root Cause Guess:`**

City field's touch target likely overlaps with the Phone Number field at initial screen render, intercepting the first tap event before the layout fully settles.

---

**`Impact & Rationale:`**

Disrupts the account setup flow — a critical first-time user experience. Unexpected focus behavior reduces trust in app responsiveness and may confuse new users during onboarding.
