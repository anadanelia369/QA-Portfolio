# ODIN-1009

Title                          : Order Edit Shows Cached Data — Updates Multiple Times
Company: ODIN
Device/Browser: Android 14 / Realme 10
Environment: UAT
Severity: Medium
Status: Done
Type: Functional

**Severity:** `Medium`**Reproducibility:** `100%`

**`Preconditions:`**

- User is logged into ODIN App
- Order Edit screen is accessible

---

**`Steps to Reproduce:`**

1. Open Order Edit screen
2. Change quantity from **10 → 20**
3. Tap **Save**
4. Re-enter Order Edit screen
5. Enter special characters in the name field
6. Tap **Save**
7. Re-enter Order Edit screen again

---

**`Actual Result:`**

Order Edit screen initially shows the first saved data. After multiple re-entries, automatic updates occur and previously edited data appears — screen refreshes multiple times before settling on the correct values.

---

**`Expected Result:`**

Order Edit screen should immediately render the latest saved data on every entry — no multiple refreshes or stale data should be displayed.

---

**`Root Cause Guess:`**

Front-end likely fetches data asynchronously without clearing the previous cache first — old state renders briefly before the fresh API response arrives, causing visible flickering and stale data display.

---

**`Impact & Rationale:`**

User may believe their edits were not saved, leading to confusion or duplicate edits. In an order management context, showing incorrect data — even briefly — undermines trust in the system and could result in incorrect order submissions.
