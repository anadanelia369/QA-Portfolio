# ODIN-1079

Title                          : Offers Tab Displays Incorrect Date Range (17–17 Instead of 17–18)
Company: ODIN
Device/Browser: Android 14 / Realme 10
Environment: UAT
Severity: Medium
Status: Done
Type: Backend

**`Preconditions:`**

- User is logged into ODIN App as Traveler
- Sender has created an Auction Order with delivery date range: March 17–18
- Trip TK001 exists and is visible in the Trips tab

---

**`Steps to Reproduce:`**

1. Open ODIN App
2. Tap **Trips**
3. Open **TK001** → View Dashboard
4. Navigate to **Offers** tab
5. Observe the delivery date displayed

---

**`Actual Result:`**
Delivery date in the Offers tab shows **17–17** instead of the full range specified by the Sender.

---

**`Expected Result:`**

Delivery date in the Offers tab should display the complete range: **17–18**, consistent with what the Sender entered during Order creation and what is correctly shown after payment in Ongoing status.

**`Notes:`**

After the Sender completes payment and the order moves to **Ongoing** status, the date range **17–18**

is correctly reflected. The bug is specific to the **Offers tab prior to payment** — suggesting the backend returns only the start date at this stage.
